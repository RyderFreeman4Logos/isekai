# 世界观数据库同步协议

每当你撰写或修改小说章节时，必须同步更新 `docs/names/` 下相关对象的词条。

## 触发条件
- 完成一章初稿后。
- 在现有章节中增加了新的重要设定或剧情进展后。

## 执行清单
1.  **识别对象**: 列出本章登场或被提及的主要角色、组织、地点。
2.  **更新经历**: 在对应的 `.md` 文件中，添加或更新“最新经历”或“当前状态”部分。
    *   *示例*: 在 `沃卡.md` 中记录他刚刚获得的道具或达成的盟约。
    *   *示例*: 在 `罗德尔王国.md` 中记录王都被毁的事件。
3.  **新建词条**: 如果出现了新的重要人物或势力，立即创建新的 `.md` 文件。
4.  **消歧义**: 如果剧情导致原有设定发生变化（如角色阵营反转），需注明变化原因并更新核心设定。

## 动机目的
- **避免遗忘**: 避免写了十章剧情，人物卡还停留在第一章的状态。
- **避免冲突**: 确保正文剧情与人物卡中的描述一致。
